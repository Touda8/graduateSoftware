# 编译输出与日志管理规范

> **目标**：确保 Visual Studio 编译产物和日志文件不污染项目主目录，所有中间产物和日志都集中在 `build/` 和 `logs/` 目录管理。

---

## 📂 目录结构规范

```
f:/project/graduateSoftware/
├── src/              # 源代码（Git 跟踪）
├── test/             # 测试代码（Git 跟踪）
├── docs/             # 文档（Git 跟踪）
├── algorithm/        # MATLAB 算法参考（Git 跟踪）
│
├── config/           # 运行时配置（.gitignore 忽略）
│   └── params.json
│
├── build/            # ⭐ 编译输出 & 中间产物（.gitignore 忽略）
│   ├── Release/
│   │   ├── *.exe / *.dll / *.lib
│   │   └── obj/      # 中间产物：.obj、.pdb、.ilk
│   ├── Debug/
│   │   ├── *.exe / *.dll / *.lib
│   │   └── obj/
│   └── old/          # 旧的 .obj 备份（cleanup.bat 创建）
│
├── logs/             # ⭐ 所有日志文件（.gitignore 忽略）
│   ├── build_*.log   # MSBuild 编译日志（build.bat 生成）
│   └── app_*.log     # 应用运行日志（Logger 输出）
│
└── *.bat             # 构建脚本
    ├── build.bat     # VS2019 编译+日志记录
    ├── cleanup.bat   # 清理主目录的旧产物
    └── run.bat       # 执行编译后的程序
```

---

## 🔄 工作流程

### 1️⃣ **第一次清理（一次性）**

如果主目录中已有旧的 `.obj` 或 `.txt` 文件：

```bash
# 运行清理脚本
cleanup.bat
```

效果：
- ✅ 所有 `*.obj` → `build/old/`
- ✅ 所有 `*.txt` → `logs/`

---

### 2️⃣ **正常编译流程（持续使用）**

```bash
# 编译 Release 配置
build.bat Release

# 或者编译 Debug 配置
build.bat Debug
```

自动行为：
- ✅ `build.bat` 自动创建 `build/Release/obj/` 和 `build/Debug/obj/`
- ✅ 创建 `logs/` 目录（若不存在）
- ✅ 编译日志自动写入 `logs/build_<TIMESTAMP>.log`
- ✅ 所有 `.obj` 输出到 `build/<Config>/obj/`，**不会污染主目录**

---

### 3️⃣ **验证 .gitignore 配置**

确保 `.gitignore` 包含以下条目：

```gitignore
# 编译产物和中间文件
build/                # 整个 build/ 目录（Release/Debug/old 及其 obj/）
logs/                 # 整个 logs/ 目录
*.obj                 # 防止遗漏的 .obj 文件
*.log                 # 防止遗漏的 .log 文件
```

**检查**：
```bash
git status
# 不应该显示任何在 build/、logs/ 或 *.obj/*.log 中的文件
```

## 📊 .vcxproj.props 配置详解

`twoProjector.vcxproj.props` 中的关键配置：

```xml
<!-- Release 配置 -->
<PropertyGroup Condition="'$(Configuration)|$(Platform)'=='Release|x64'">
  <OutDir>$(ProjectDir)build\Release\</OutDir>
  <IntDir>$(ProjectDir)build\Release\obj\</IntDir>
</PropertyGroup>

<!-- Debug 配置 -->
<PropertyGroup Condition="'$(Configuration)|$(Platform)'=='Debug|x64'">
  <OutDir>$(ProjectDir)build\Debug\</OutDir>
  <IntDir>$(ProjectDir)build\Debug\obj\</IntDir>
</PropertyGroup>
```

| 变量 | 含义 | 示例 |
|------|------|------|
| `$(ProjectDir)` | 项目根目录 | `F:\project\graduateSoftware\` |
| `OutDir` | 最终输出目录（.exe/.dll/.lib） | `build\Release\` |
| `IntDir` | 中间文件目录（.obj/.pdb） | `build\Release\obj\` |

## 🛠️ 常见问题

### Q1：为什么我的 .obj 文件还在主目录？

**A**：说明 `.vcxproj.props` 的 `IntDir` 配置未生效。可能原因：
1. 没有运行 `cleanup.bat` 清理旧文件
2. Visual Studio 缓存（关闭 Visual Studio，删除 `.vs/` 目录，重新打开）
3. `.vcxproj.props` 没有被 `.vcxproj` 导入

**解决方案**：
```bash
# 清理旧文件
cleanup.bat

# 清理 VS 缓存（关闭 VS 后运行）
rmdir /s /q .vs

# 重新打开 Visual Studio 并编译
```

---

### Q2：日志文件太多，怎么查看？

**A**：最新的日志文件：
```bash
cd logs
dir /s /a /b *.log | sort /r | head -1    # 最新文件
type <最新文件>                           # 查看内容
```

---

### Q3：可以手动删除 `build/old/` 中的文件吗？

**A**：✅ **可以**。`build/old/` 目录只用于备份，任何时候都可以删除。

```bash
rmdir /s /q build\old
mkdir build\old     # 下次清理时自动重建
```
