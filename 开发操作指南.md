## 一、系统概述

### 1.1 项目简介

本项目是一套**远心单目双光栅三维重建测量系统**，基于双投影仪结构光原理，采用三频外差相位展开算法对工件进行高精度三维重建，并通过点云分析完成 BGA 焊球及 QFP 引脚的共面度测量。

| 项目信息 | 内容 |
|---------|------|
| 工程名称 | graduateSoftware |
| 主窗口类 | `twoProjectorClass : QMainWindow` |
| UI 文件 | `twoProjector.ui`（禁止手动修改） |
| 语言标准 | C++17（`/std:c++17`） |
| 构建工具 | Visual Studio 2019 / MSBuild，v142 工具集，x64 |

### 1.2 多 Agent 并行开发模式

本项目采用 **GitHub Copilot 多 Agent 并行开发模式**，由 7 个命名 Agent 协同完成：

- **总指挥** 负责任务调度，不直接写代码
- **6 个专项 Agent** 分三阶段并行执行，每阶段内两个 Agent 同步开工
- 所有 Agent 定义在 `.github/agents/` 目录，在 VS Code Copilot Chat 中通过下拉菜单选择

## 三、如何在 VS Code 中选择并激活总指挥

### 核心原则：只选一个 Agent，永不手动切换子 Agent

> **每次开发会话，只在下拉菜单中选择「总指挥 (Chief Orchestrator)」，
> 由总指挥自动通过 runSubagent 工具并行或串行调用所有子 Agent。
> 你只需要向总指挥发送指令，无需关心内部由哪个子 Agent 执行。**

### 3.1 打开 Copilot Chat 并选择总指挥

1. 按 `Ctrl+Alt+I` 打开 GitHub Copilot Chat 面板
2. 点击输入框**左侧下拉菜单**（默认显示 "Ask"）
3. 选择 **`总指挥 (Chief Orchestrator)`**
4. 在同一对话窗口中完成整个阶段的所有工作

```
[ 总指挥 (Chief Orchestrator) ▼ ]   输入你的指令...
```

> 若列表中没有「总指挥」：确认工作区根目录是 `F:\project\graduateSoftware`，
> 然后重启 VS Code 重新加载 `.github/agents/` 目录。

### 3.2 总指挥自动调度原理

```
你（用户）输入一条指令
        │
        ▼ 总指挥接收
        │
        ├─ 并行 runSubagent("需求解析师") ─────┐
        │                                      │ 同时运行
        └─ 并行 runSubagent("工程配置师") ─────┘
                        │
                        ▼ 两个子 Agent 均完成后
                门控检查 → 汇报给你
                        │
        ├─ 并行 runSubagent("UI 开发师") ──────┐
        │                                      │ 同时运行
        └─ 并行 runSubagent("算法翻译师") ─────┘
                        │
                        ▼ ... 以此类推
```

### 3.3 一次开发的对话次数

| 阶段 | 你发送的消息数 | 总指挥内部调度 |
|------|-------------|-------------|
| 启动前自检 | **1 条** | 总指挥自检环境 |
| Phase 1 | **1 条** | 并行调用需求解析师 + 工程配置师 |
| Phase 2 | **1 条** | 并行调用 UI 开发师 + 算法翻译师 |
| Phase 3 | **1 条** | 并行调用异常防护师 + 测试验收师 |
| 修复问题 | **1 条/问题** | 总指挥派发给对应子 Agent |

**整个项目从零到完成，理论上只需 4~6 条消息。**

### 3.4 什么时候才需要手动选择子 Agent？

仅在以下情况下才允许绕过总指挥直接选子 Agent：
- 总指挥汇报某子 Agent 无法自动启动，要求你手动触发
- 你只需要对**单个已完成文件**做小修改（< 5 行）
- 总指挥明确告知需要手动介入

**除上述情况外，始终选总指挥。**

## 五、Phase 1 — 总指挥自动并行执行

**你只需发送一条消息（见第八章完整 Prompt）。总指挥会自动：**

### 总指挥内部并行调度流程

```
总指挥收到 Phase 1 指令
        │
        ├── [并行 A] runSubagent("需求解析师")
        │       任务：读取 twoProjector.ui + global_description.md
        │       产物：docs/forbidden_list.md
        │             docs/ui_algorithm_binding.md   ← UI控件与算法函数对应表
        │             docs/task_list.md
        │             src/common/interfaces.h         ← 跨模块接口定义
        │
        └── [并行 B] runSubagent("工程配置师")
                任务：扫描 F:\project\Envlib 所有 .lib，生成工程文件
                产物：twoProjector.vcxproj.props      ← 库路径属性表
                      .gitignore
                      build.bat                       ← 一键构建脚本
                      (Git 初始化)
```

### 门控检查（由总指挥自动执行，你无需操作）

总指挥在进入 Phase 2 前自动验证：

| 检查项 | 验证方式 |
|--------|---------|
| `src/common/interfaces.h` 存在 | 文件系统检查 |
| `interfaces.h` 包含 IDecoder/IHeightCalc/IFilter/IMeasure | grep 检查 |
| `docs/ui_algorithm_binding.md` 非空 | 行数 > 10 |
| `twoProjector.vcxproj.props` 存在 | 文件系统检查 |
| `build.bat` 可运行 (exit code 0) | 终端执行验证 |

任一失败 → 总指挥向你报告阻塞原因，等待确认后再继续。

### 文件操作规范（子 Agent 内部遵循，你无需关心）
- 每个文件先创建骨架（≤ 30 行），再逐节插入实现
- 每次 replace 不超过 50 行，每 3 次插入后读文件验证

## 九、常见问题与约束速查

### Q0：为什么必须用总指挥，不能直接启动子 Agent？

总指挥通过 `runSubagent` 工具在**同一轮对话**内并行调用多个子 Agent，
避免你手动多次切换 Agent、多次确认上下文。直接启动子 Agent：
- 无法并行（需要等一个完成再手动启动另一个）
- 缺少门控验证（没有自动检查前置条件）
- 对话次数更多，上下文容易丢失

**例外**：总指挥报告某子 Agent 无法自动触发时，才手动选择该子 Agent。

### Q1：我误操作修改了 capWidget，怎么恢复？
```bash
git diff twoProjector.ui       # 查看改动
git checkout twoProjector.ui   # 恢复文件
```

### Q2：Agent 说找不到某 .lib 文件怎么办？
```powershell
Get-ChildItem "F:\project\Envlib" -Recurse -Filter "*.lib" |
  Select-Object Name, FullName | Format-Table -AutoSize
```
将输出发给**总指挥**，让总指挥派发给工程配置师更新 `.vcxproj.props`。

### Q3：Qt `connect` 报编译错误 "no matching signal"？
向总指挥发送：
```
MainWindow.cpp 中第 [N] 行的 connect 编译失败，错误信息：[粘贴错误]。
请读取该行的完整上下文并修正信号签名（只改这一处，单次 ≤ 5 行）。
```

### Q4：算法翻译后相位值域超出 [-π, π]？
向总指挥发送：
```
PhaseDecoder.cpp 中 calcPhaseMatrix() 相位值超范围，请读取该函数完整实现，
检查 atan2 之后是否错误地乘以了 2π，并精确修正（读上下文 → 单次 replace ≤ 10 行）。
```

### Q5：VTK 窗口不显示点云？
向总指挥发送：
```
VtkWidget::updateCloud() 末尾未调用 ResetCamera/Render，
请读取该函数完整实现，在正确位置补充这两行调用（先读上下文 ≥ 5 行再替换）。
```

### 约束速查表

| 限制项 | 规则 |
|--------|------|
| Agent 选择 | 始终选「总指挥」，子 Agent 由总指挥自动调度 |
| 禁区 Tab | `capWidget` 所有子控件绝对禁止触碰 |
| 库目录 | `F:\project\Envlib` 只读，禁止修改任何文件 |
| 跨线程 UI | 工作线程回调必须通过 `Qt::QueuedConnection` 信号传回 |
| create_file | 单次 ≤ 30 行（仅骨架），禁止一次性写入全量实现 |
| replace_string | 单次新内容 ≤ 50 行；每 3 次插入后读文件验证 |
| 全局变量 | 禁止非 `const` 全局变量；Logger/Config 用单例 |
| 主线程 | 禁止执行 > 5ms 的操作（图像/点云运算在线程池）|
